# Спецификации компонентов дизайн-системы

Документация компонентов дизайн-системы на стеке **PrimeUI** (PrimeNG · PrimeReact · PrimeVue · React Native 📱). Каждая спецификация разбита на две секции: **🎨 для дизайнеров** (анатомия, варианты, размеры, состояния, токены) и **💻 для разработчиков** (API, примеры, кастомизация, отличия фреймворков).

## Структура

- [`COMPONENT-SPEC-TEMPLATE.md`](./COMPONENT-SPEC-TEMPLATE.md) — переиспользуемый шаблон. Копируйте и заполняйте плейсхолдеры `{{...}}`.
- [`examples/button.md`](./examples/button.md) — заполненный пример (`<Button>`).

## Источники данных

| Секция | Источник |
|---|---|
| 🎨 Для дизайнеров | Figma «UI Kit (DS) v2.0»: оси variant-свойств, размеры, токены |
| 🎨 Токены | `angular-ui-kit/семантическая модель/` (`DESIGNER-GUIDE.md`, `tokens.semantic-*.json`) |
| 💻 Для разработчиков | `angular-ui-kit` (PrimeNG), `vue-ui-kit-3` — API, stories, тесты |

## Как добавить новый компонент

1. Скопируйте `COMPONENT-SPEC-TEMPLATE.md` в `examples/<component>.md`.
2. Дизайн-секцию заполните из Figma (component set → variant-оси, размеры, привязки токенов).
3. Секцию разработчиков — из репозиториев (компонент, `*.stories.*`, `*.test.*`).
4. Расхождения дизайн ↔ код фиксируйте в секции 2.8.
